# Showcase

*  **This project itself uses Konture** - Check out the <a href="https://github.com/baole/konture/tree/main/konture-test" target="_blank" rel="noopener noreferrer">konture-test</a> module for examples of how to write tests using Konture.
*   **<a href="https://github.com/baole/konture/tree/main/showcases/sample-gradle" target="_blank" rel="noopener noreferrer">🛠️ Showcase A: Multi-Module Gradle (`sample-gradle`)</a>** - A standard Kotlin multi-project Gradle setup with a dedicated test execution module, including source-set-scoped test-code usage rules.
*   **<a href="https://github.com/baole/konture/tree/main/showcases/sample-maven" target="_blank" rel="noopener noreferrer">📦 Showcase B: Multi-Module Maven (`sample-maven`)</a>** - Set up in an Apache Maven ecosystem, utilizing the Maven compiler and custom multi-module configurations.
*   **<a href="https://github.com/baole/nowinandroid/tree/main/konture-test" target="_blank" rel="noopener noreferrer">Now in Android (NiA)</a>** - Google's reference modular Android application using a dedicated `:konture-test` module.
*   **<a href="https://github.com/baole/kotlinconf-app/tree/main/konture-test" target="_blank" rel="noopener noreferrer">KotlinConf Companion App</a>** - A full-stack Kotlin Multiplatform (KMP) client application.
*   **<a href="https://github.com/baole/ktor-arrow-example/tree/main/konture-test" target="_blank" rel="noopener noreferrer">Ktor-Arrow Backend Example</a>** - A production-grade backend demonstrating SQLDelight leakage-prevention and HTTP layer isolation rules.


If your project is using Konture, please consider adding it to this list by submitting a pull request to this file.
